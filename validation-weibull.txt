2.1  PRIMARY — Probability-Plot Initial Guess (censored data)
-------------------------------------------------------------
Location: weibullMLE(), lines ~1930-1972

CHANGE DESCRIPTION:
  When `observations.some(o => o.type === 'right')` is true, the code now
  uses Herd-Johnson adjusted ranks (medianRank / Hazen formula) to compute
  Weibull probability-plot points, then fits OLS in linearised space:

    x = ln(t),   y = ln(-ln(1 - F))
    slope  = β₀
    η₀     = exp(-intercept / β₀)

  For complete data (no right-censored observations), the Dubey (1967)
  estimator β₀ ≈ 1.283 / σ_ln is retained.

  A try/catch block wraps the probability-plot path; if it fails (e.g.,
  fewer than 2 plot points), it falls back to Dubey.

MATHEMATICAL CORRECTNESS:
  The Weibull CDF linearisation is:
    ln(-ln(1-F)) = β·ln(t) - β·ln(η)
  Therefore slope = β and intercept = -β·ln(η), giving η = exp(-b/β).
  This is textbook and correct (Abernethy 2006, ch.3; Nelson 1982, ch.4).

  The Herd-Johnson increment (already verified in v8 as correct) is reused
  correctly here. The medianRank / Hazen formula F = (j-0.5)/N is
  appropriate and consistent with the rest of the codebase.

MULTI-START INTERACTION:
  The probability-plot estimate β₀/η₀ becomes the first of four starting
  points in the multi-start Nelder-Mead. The other three starts (β₀·0.5,
  β₀·2.0, and β=1) are retained unchanged. This is correct and provides
  additional robustness. The best of all four starts is used.

  When called with useMultiStart=false (during the 3P grid scan), the
  probability-plot path is still executed to compute β₀/η₀, but only a
  single Nelder-Mead call is made from that starting point. This is
  appropriate and consistent with the speed objective of the grid scan.

EDGE CASE — interval-only censoring:
  If the dataset has only exact failures and interval-censored observations
  (no `type === 'right'` entries), `hasCensoring` evaluates to false and
  the code falls back to Dubey. This is a correctness gap documented as
  NV1/NV2 in Section 4.

VERDICT: CORRECTLY IMPLEMENTED. The change achieves its stated goal: for
right-censored datasets the MLE starts much closer to the optimum, reducing
the risk of premature convergence in flat likelihood surfaces.

2.4  M3 — Golden-Section Refinement for 3P Regression (CCO)
-------------------------------------------------------------
Location: weibullRegression3(), lines ~2160-2194

Implementation verified:
  1. Coarse 400-point grid runs first, identifying bestGamma.
  2. Golden-section search bracket: [bestGamma - step, bestGamma + step]
     where step = minT/400 (one grid spacing).
  3. Objective: maximise R² in linearised Weibull plot.
  4. 20 golden-section iterations: resolution ≈ (2·step)·φ^{-20} ≈ 5×10⁻⁹
     relative to the bracket width. This is adequate.
  5. Final evaluation at refined γ updates bestBeta, bestEta, bestR2.
  6. A silent catch preserves the coarse-grid result if the refinement
     fails, which is appropriate defensive programming.

The golden-section ratio gr = (√5-1)/2 ≈ 0.61803 is correctly implemented.
The update rule (if fcG > fdG: gsB=gsD … else gsA=gsC …) correctly
maximises R² (i.e., searches for the maximum, not the minimum).

VERDICT: CORRECTLY IMPLEMENTED. This mirrors the golden-section approach
already present in the 3P MLE path, achieving resolution below 1e-8 in γ
from approximately 20 function evaluations. The computational overhead is
negligible.


2.5  M4 — Tighter Lower Bound in negLogLikGeneral
--------------------------------------------------
Location: negLogLikGeneral(), line ~1894

OLD: `if (beta <= 0.01 || eta <= 0.01) return 1e12;`
NEW: `if (beta <= 1e-4 || eta <= 1e-4) return 1e12;`

VERDICT: CORRECTLY IMPLEMENTED. The tighter bound of 1e-4 reduces the
risk of premature rejection of valid parameter values during optimizer
exploration for datasets with very small time values or extreme β. The
penalty value 1e12 remains appropriate.

RESIDUAL NOTE: The multi-start array still contains `Math.max(0.01, eta0 * 0.5)`
(line ~1983) as a floor for η in starting points. This is now inconsistent
with the 1e-4 bound but has no practical effect: no realistic starting
η₀ would yield η₀·0.5 < 0.01, making the 0.01 floor in starting points
cosmetically mismatched but functionally irrelevant.

2.8  O1 — Warning When γ Converges Near min(t)
-----------------------------------------------
Location: weibullMLE3(), lines ~2118-2121

Implementation:
  `if (bestGamma > 0.99 * minT) showToast('⚠️ 3P: γ converged very close
   to min(t) …', 9000);`

VERDICT: CORRECTLY IMPLEMENTED. The threshold 0.99·min(t) (i.e., within
1% of min(t)) is a reasonable heuristic. The warning text correctly
identifies the pathological case (unbounded likelihood as γ→min(t) when
β<1) and recommends 2P MLE.

NOTE: The condition checks `bestGamma > 0.99 * minT` after the
golden-section refinement, which operates in the range [0, 0.9999·minT].
Since the golden-section upper bound is 0.9999·minT, any γ in
(0.99·minT, 0.9999·minT) will trigger the warning. For a dataset where
the true 3P γ is legitimately close to min(t), this warning may be a
false positive. However, the warning is advisory (not blocking) and is
appropriate given the numerical risk.

4. NEW ISSUES DISCOVERED IN v9
================================================================================

NV1 — Probability-Plot Initialiser Ignores Interval-Censored Observations
---------------------------------------------------------------------------
Severity: MINOR
Location: weibullMLE(), line 1937

The `hasCensoring` flag is:
  `const hasCensoring = observations.some(o => o.type === 'right');`

This is true only when there are right-censored (suspension) observations.
Datasets containing interval-censored observations but NO right-censored
observations will evaluate `hasCensoring = false` and use the Dubey
estimator, even though interval censoring also distorts the log-variance
used by Dubey and could benefit from the probability-plot approach.

Example: failures=[100,150,200], intervals=[250-300, 350-400]
→ hasCensoring = false → Dubey initialiser used
→ Dubey uses only exact failure times: σ_ln computed from 100, 150, 200
→ The interval observations are ignored in the initial guess, potentially
  producing a poor β₀ when the true β would be influenced by the
  interval data.

The probability-plot path only uses right-censored suspensions anyway
(via `suspTimesInit`), so extending `hasCensoring` to include interval
observations would trigger the probability-plot path but the interval
data would still not be incorporated into the adjusted ranks. The
practical benefit is limited, but the semantic inconsistency is notable.

RECOMMENDATION: Change the flag to:
  `const hasCensoring = observations.some(o => o.type === 'right' ||
                                               o.type === 'interval');`
This ensures the probability-plot path is used whenever any censoring
is present, which is statistically consistent with the intent of the
feature.


NV2 — Inconsistent Fallback Floor in Multi-Start vs. New Lower Bound
---------------------------------------------------------------------
Severity: OBSERVATION
Location: weibullMLE(), line ~1983

After M4 lowered the negLogLikGeneral lower bound to 1e-4, the multi-start
array still applies a 0.01 floor to starting η values:

  `[Math.min(20, beta0 * 2.0), Math.max(0.01, eta0 * 0.5)]`

If eta0 is on the order of 0.02 (small-time dataset), then eta0*0.5 = 0.01
which is valid — but the floor of 0.01 means the optimizer cannot explore
η values in (1e-4, 0.01) from this starting point. Since the negLogLik
now accepts β,η > 1e-4, a starting point of η=0.01 could still miss the
MLE if the true η is in (1e-4, 0.01).

In practice this only affects datasets with very small absolute time values
(sub-millisecond or similar), and the other three starts (which have no
such floor) would likely find the correct basin. The inconsistency is
cosmetic for typical engineering datasets.

RECOMMENDATION: Change `Math.max(0.01, eta0 * 0.5)` to
`Math.max(1e-4, eta0 * 0.5)` for full consistency with M4.

5.9  3P MLE Grid + Golden-Section (pre-existing)
  The 3P grid now calls weibullMLE(sh, 0, 3, false) at each grid point
  with useMultiStart=false. With the v9 change, each of these 200 calls
  will now execute the probability-plot initialiser (since the shifted
  observations `sh` may contain right-censored entries when there are
  suspensions in the original data).

  INTERACTION CONCERN: When `useMultiStart=false`, the probability-plot
  path computes adjusted ranks and runs a linear regression for EACH of
  the 200 grid points. This adds ~200 × (rank computation + regression)
  operations per 3P MLE call. For a dataset with n suspensions, each rank
  computation is O(n log n). For large n (e.g., 1000+ observations), this
  could increase 3P MLE runtime by a meaningful amount.

  Quantitative estimate: For n=100 total observations, each adjusted-rank
  computation ≈ 1 ms; 200 grid points × 1 ms = 200 ms additional overhead.
  For n=1000, this could reach 2-5 seconds additional. The golden-section
  phase (25 iterations) adds 25 more such calls.

  This is a new PERFORMANCE OBSERVATION. For datasets with n<200 it is
  negligible. For larger n with censored data and 3P MLE, the user may
  notice increased computation time. The current async chunking does NOT
  apply to the 3P grid (it runs synchronously inside a for loop), so
  the UI could freeze for 2-5 seconds on large censored datasets.

  This was not present in v8 because the Dubey initialiser (O(n)) was
  used unconditionally.

  RECOMMENDATION: Add `useMultiStart=false` path to skip the probability-
  plot initialiser and always use Dubey when called from the 3P grid,
  since the multi-start benefit is already bypassed and the grid scan
  does not require a high-quality start — only a consistent one:

    if (hasCensoring && useMultiStart) {
        // probability-plot initialiser
    } else {
        // Dubey initialiser (fast, consistent)
    }

7. RECOMMENDATIONS
================================================================================

Priority 1 — Address NV3 (Performance regression, 3P + censored):
  Guard the probability-plot path with `useMultiStart`:

    const hasCensoring = observations.some(o => o.type === 'right');
    if (hasCensoring && useMultiStart) {
        // probability-plot initialiser (v9 path)
    } else {
        // Dubey initialiser (used for grid scans and complete data)
    }

  This preserves the full benefit of the new initialiser for the final
  quality MLE (where useMultiStart=true) while keeping grid scans fast.
  Effort: 2-line change. Impact: eliminates the O(n log n × 200) overhead
  during the 3P grid scan.

Priority 2 — Address NV1 (interval-only censoring):
  Extend hasCensoring to include interval-censored observations:

    const hasCensoring = observations.some(
        o => o.type === 'right' || o.type === 'interval');

  Note: even with this change, interval observations are not incorporated
  into the adjusted-rank computation (the probability-plot only uses exact
  failures and right-censored suspensions). The benefit is purely that
  the flag correctly signals "censoring is present" for interval-only
  datasets, and the fallback to Dubey is avoided.

Priority 3 — Address MAJ1 (Bootstrap interval resampling):
  For interval-censored observations in bootstrap resamples, sample a
  latent failure time from the truncated Weibull within [left, right]:

    const u = Math.random();
    const Fl = weibullCDF(left, etaHat, betaHat, gamHat);
    const Fr = weibullCDF(right, etaHat, betaHat, gamHat);
    const Fmix = Fl + u * (Fr - Fl);
    const tLat = gamHat + etaHat * Math.pow(-Math.log(1 - Fmix), 1/betaHat);
    return { type: 'exact', time: tLat };

  This is statistically correct and adds two lines of code per resample.

Priority 4 — Address NV2 (Multi-start floor inconsistency):
  Change `Math.max(0.01, eta0 * 0.5)` to `Math.max(1e-4, eta0 * 0.5)`.
  Effort: 1-character change. Low priority.

Priority 5 — Address MAJ3 (Number parser):
  Add an explicit locale selector (US/EU) or display parsed values
  back to the user before analysis for verification.

Priority 6 — Address M2 formula (Regression CIs):
  Replace β ± z·β/√n with SE(β̂) = √(σ²ε/SSxx) from OLS theory.
  SSxx and σ²ε = SSres/(n-2) are already available inside linearRegression().
  Expose them and use them for the CI computation.

8. CONCLUSION
================================================================================

Version 9 delivers nine targeted improvements to the v8 codebase, all of
which have been verified as mathematically correct and correctly implemented.
The primary algorithmic enhancement — using the Weibull probability plot with
Herd-Johnson adjusted ranks to initialise the MLE optimiser when censored
data is present — is technically sound, well-documented, and follows the
approach recommended by Abernethy (2006) and Nelson (1982) for censored
reliability datasets.

The three remaining major issues (MAJ1-3) are unchanged from v8. MAJ1
(bootstrap interval resampling) remains the highest-priority unresolved
defect and should be the focus of the next development cycle.

The newly discovered performance interaction (NV3) is a direct consequence
of the v9 probability-plot initialiser being called inside the 3P grid
scan: a one-line guard (`hasCensoring && useMultiStart`) eliminates this
overhead with no loss of accuracy. This fix should accompany any v10
release.

The tool continues to meet the quality bar of professional-grade reliability
analysis software for its intended scope. With the NV3 performance fix and
the MAJ1 bootstrap improvement, the tool would be suitable for production
use with interval-censored dominated datasets as well.
