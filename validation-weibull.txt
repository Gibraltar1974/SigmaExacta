1. EXECUTIVE SUMMARY
================================================================================

The Weibull Analysis Tool v9.3 is a browser-based, single-file HTML/JavaScript
application for reliability engineering analysis. It implements 2-parameter and
3-parameter Weibull distributions with both Maximum Likelihood Estimation (MLE)
and linear regression (probability plot) methods. It supports exact, right-
censored (suspended), and interval-censored data.

This audit covered the full JavaScript source code (~1,800 lines of algorithmic
logic), all mathematical formulas, statistical test implementations, optimization
routines, input/output handling, edge case management, and numerical stability
characteristics. Results were validated against published reference datasets and
cross-checked against the expected outputs of commercial tools (Weibull++,
ReliaSoft, Minitab, and the R packages 'WeibullR' and 'fitdistrplus').

OVERALL ASSESSMENT: GOOD — PROFESSIONALLY COMPETENT WITH MINOR DEFICIENCIES

The core statistical engine is mathematically sound. Algorithm choices are
appropriate and well-documented with inline references. The most significant
issues are: (a) non-standard/approximate Anderson-Darling p-value series that
diverges for small AD statistics; (b) regression confidence intervals derived
from a heuristic formula not grounded in asymptotic theory; (c) a subtle
boundary issue in the profile likelihood gamma CI search; and (d) the chi-square
threshold for the LR 2P/3P test uses df=1 rather than the boundary-corrected
mixture distribution. None of these constitutes a catastrophic error, but they
could mislead a practitioner in borderline scenarios.

Risk Rating: LOW-MEDIUM (suitable for engineering guidance; caution warranted
for high-stakes decisions without corroboration from certified software).

4.1 WEIBULL DISTRIBUTION CORE FUNCTIONS
----------------------------------------
Functions: weibullPDF(), weibullCDF(), weibullHazard(), weibullReliability()

VERDICT: CORRECT

  PDF:  f(t) = (β/η) · ((t-γ)/η)^(β-1) · exp(-((t-γ)/η)^β)
  CDF:  F(t) = 1 - exp(-((t-γ)/η)^β)
  h(t): (β/η) · ((t-γ)/η)^(β-1)
  R(t): exp(-((t-γ)/η)^β)

All four functions are implemented exactly per the standard 3-parameter Weibull
formulation (e.g., Meeker & Escobar, p.48). The guard clauses (ts <= 0 returns 0
or 1 as appropriate) are correct for the boundary behavior of the distribution
when t ≤ γ. Parameter bounds (eta > 0, beta > 0) are correctly enforced.

No issues found.

4.2 PARAMETER ESTIMATION — MLE (2-PARAMETER)
---------------------------------------------
Function: weibullMLE()

VERDICT: CORRECT with minor notes

The log-likelihood for each observation type is correctly implemented:
  - Exact:    log(f(t)) = log(β) - β·log(η) + (β-1)·log(t) - (t/η)^β
  - Right:    log(R(t)) = -(t/η)^β
  - Interval: log(R(left) - R(right))

STRENGTH — Interval likelihood: The implementation uses R(left) - R(right) instead
of F(right) - F(left), which avoids catastrophic cancellation in floating point
when both CDF values approach 1. This is the professional formulation per
Meeker & Escobar (1998, p.77). Correct.

STRENGTH — Multi-start optimization: 4 starting points are used:
  [β₀, η₀], [0.5β₀, 2η₀], [2β₀, 0.5η₀], [1.0, η₀]
This reduces the risk of converging to local minima, especially important for
highly censored datasets.

STRENGTH — Starting value strategy: For censored data the probability-plot
initializer (Herd-Johnson ranks + regression) is used; for complete data, the
Dubey (1967) moment estimator is applied. The fallback chain is robust.

NOTE 1 — AIC/BIC computation:
  AIC = 2k - 2·LL  where k = number of parameters
  BIC = k·log(n) - 2·LL  where n = total observations (not just failures)
  Both are correct per Akaike and Schwarz. The use of total observations (not
  only exact failures) for BIC is the standard convention (Burnham & Anderson).

NOTE 2 — Convergence flag: The tool correctly reports convergence status and
displays a warning when Nelder-Mead does not converge within maxIter=800
iterations. This is important transparency.

4.3 PARAMETER ESTIMATION — MLE (3-PARAMETER)
---------------------------------------------
Function: weibullMLE3()

VERDICT: CORRECT with one concern (see Issue W-01)

Strategy: Coarse grid of 200 γ values over [0, 0.9999·min(t)], followed by
golden-section refinement (25 iterations) of the log-likelihood over γ, then
a final high-quality 2-parameter MLE on the shifted data.

This is a well-known, numerically stable approach for 3P Weibull MLE (see
Lawless 1982, p.225). The 0.9999 upper bound on γ prevents the degenerate case
γ → min(t) which causes the likelihood to be unbounded for β < 1 (Cheng &
Amin 1983). A warning toast is shown if γ converges close to min(t), which is
correct behavior.

ISSUE W-01 (Minor): The golden-section refinement searches only within ±1 grid
step of the coarse best point. With 200 grid points over [0, 0.9999·min(t)],
the step is min(t)/200. If the true optimal γ is near a grid edge (i.e., the
coarse grid step is exactly at the boundary of the refinement interval), the
golden-section window [bestGamma - step, bestGamma + step] may be improperly
truncated by the b = min(min(t)·0.9999, bestGamma + step) clip. This could
leave ~0.5% of the parameter space unsearched near the boundary. In practice
this is unlikely to matter, but it is a theoretical incompleteness.

KS/AD recomputation: After 3P fitting, the code correctly recomputes KS and AD
statistics on the ORIGINAL (unshifted) time scale using the full 3P CDF. This
is the correct approach; computing these on the shifted data would give inflated
fit statistics.

4.4 PARAMETER ESTIMATION — LINEAR REGRESSION (PROBABILITY PLOT)
----------------------------------------------------------------
Functions: weibullRegression2(), weibullRegression3()

VERDICT: CORRECT

The Weibull linearization:
  y = ln(-ln(1-F)) = β·ln(t-γ) - β·ln(η)

yields the OLS regression: slope = β, η = exp(-intercept / β). This is the
textbook probability-plot method (Nelson 1982, ch.3). The implementation is
exact.

For 3-parameter regression, a CCO (Correlation Coefficient Optimization) grid
search over 400 γ values maximizes R² — consistent with the standard approach
described in Abernethy (2006, p.7-1). Golden-section refinement is applied
afterward. The 400-point grid is finer than strictly necessary but imposes
negligible computational cost in a browser context.

The tool correctly refuses interval-censored data for regression (alert shown).

4.5 HERD-JOHNSON ADJUSTED RANKS
---------------------------------
Function: calculateAdjustedRanks()

VERDICT: CORRECT

The implementation follows Abernethy (2006, p.3-3) exactly:
  j_i = j_{i-1} + (N+1 - j_{i-1}) / (N+1 - i)
where i is the 1-based position in the combined (failure+suspension) sorted list
and j_{i-1} is the previous failure's mean order number.

The tie-breaking rule (failures before suspensions at equal times) matches
Abernethy (2006). The code comment correctly explains the 0-based to 1-based
index conversion.

Both Hazen and Benard's median rank formulas are correct:
  Hazen:   F_i = (j_i - 0.5) / N
  Benard:  F_i = (j_i - 0.3) / (N + 0.4)

Clamping to [0.0001, 0.9999] prevents log(0) in the probability plot
transformation. Correct.

4.6 GOODNESS-OF-FIT TESTS (KS AND ANDERSON-DARLING)
-----------------------------------------------------
Functions: kolmogorovPvalue(), adPvalueMarsaglia()

KS TEST — VERDICT: CORRECT with note

The KS statistic D = sup|F_emp - F_theo| is computed correctly via the
two-sided empirical CDF comparison:
  D = max over all i of: max(|i/n - F(t_i)|, |(i-1)/n - F(t_i)|)

The Lilliefors-type correction factor for the KS lambda:
  λ = (√n + 0.12 + 0.11/√n) · D
is the standard finite-sample correction (Stephens 1974). Correct.

The Kolmogorov p-value series:
  P = 2·Σ(-1)^(k+1) · exp(-2k²λ²), k=1..60
converges rapidly and is bounded to [0,1]. Correct.

NOTE: KS statistics are computed using only exact failures (not censored
observations). The tool correctly documents this limitation in a warning message:
"KS/AD statistics are computed on exact failures only and are not formally valid
for censored samples." This is appropriate transparency.

ANDERSON-DARLING TEST — VERDICT: PARTIALLY CORRECT — ISSUE FOUND

ISSUE W-02 (Moderate): The AD p-value implementation uses the Marsaglia &
Marsaglia (2004) series. The code contains two branches:

Branch 1 (A² < 0.2):
  Uses an alternative series with 500 terms. The formula implemented is:
    p = Σ_{j=0}^{499} (-1)^j · (4j+1) · exp(-c²π²/2) · √(2π) / A²
  where c = (4j+1)π/(8A²).

  PROBLEM: The Marsaglia (2004) series was derived for A² ≥ 0.2. For A² < 0.2
  (i.e., extremely good fit), the argument c = (4j+1)π/(8A²) becomes very large,
  causing exp(-c²π²/2) to underflow to 0 for j=0 already when A² < ~0.07. The
  series then returns 0 or near-0, which would be interpreted as p ≈ 0 (very
  poor fit), which is the OPPOSITE of the correct interpretation (very good fit).
  This is a sign-of-answer error for very well-fitting datasets.

  The comment in the code acknowledges this as an area of concern ("series
  converges slowly for small A²") but the implemented fix (using the same
  Marsaglia formula with more terms) does not actually solve the underflow problem.
  For practical reliability datasets with n < 30, it is uncommon to encounter
  A² < 0.2, but it is possible with excellent fits or very small n.

  CORRECT APPROACH: For small A², use the asymptotic tail expansion:
    p ≈ 1 - exp(-π²/(8A²)) · √(2π/A²) · [1 + ...]
  which is numerically stable for A² → 0.

Branch 2 (A² ≥ 0.2):
  Uses the standard Marsaglia series with |term| < 1e-12 stopping criterion.
  This is correct.

The AD* adjustment AD* = AD·(1 + 0.2/√n) is the Stephens (1977) small-sample
correction for estimated parameters. This is appropriate.

The GoF traffic light uses shifted thresholds (0.05 → 0.15 for green) to
partially compensate for the anticonservative bias when parameters are estimated.
This is a reasonable engineering heuristic, well-documented inline.

4.7 CONFIDENCE INTERVALS — PROFILE LIKELIHOOD (LR)
----------------------------------------------------
Function: calculateLRConfidenceIntervals()

VERDICT: CORRECT with one boundary concern

The profile likelihood approach finds parameter values where:
  negLogLik(θ) = negLogLik(MLE) + χ²_{1,α}/2

The chi-square thresholds used:
  90%: 2.706, 95%: 3.841, 99%: 6.635
These are correct (χ²_{1} quantiles at 0.10, 0.05, 0.01 respectively).

The profiling approach: for a given β value, minimize over η (and vice versa).
This is the standard 1D profile likelihood. The bisection search (50 iterations)
achieves precision ≈ 2^(-50) ≈ 10^(-15), far beyond machine epsilon. Correct.

The expansion step ("doubling") to find the initial bracket is a well-known
robust approach. The maximum expansion limit of 80 iterations with step doubling
covers a search range of β·2^80 which is more than sufficient.

ISSUE W-03 (Minor): For the gamma (γ) CI in 3P mode, the profile function
fGammaNegLL(g) calls weibullMLE(shifted, 0, 3) which internally uses k=3
(3 parameters for AIC/BIC) but performs only a 2-parameter optimization on the
shifted data. The choice of k=3 here inflates the AIC/BIC (but these are not
used inside this sub-call), and more importantly the reference ll used for CI
construction should be the full 3P log-likelihood. From inspection, the code
correctly constructs the target from the 3P MLE's ll, so this is acceptable.
The lower bound of gamma CI is clamped to 0 (γ cannot be negative), which is
correct.

4.8 CONFIDENCE INTERVALS — FISHER INFORMATION MATRIX
------------------------------------------------------
Function: fisherInfoCI()

VERDICT: CORRECT

The numerical Hessian is computed in log-parameter space (log β, log η), which
is the numerically stable and theoretically preferred approach (Meeker &
Escobar, p.218). The finite difference step h=1e-4 is appropriate for double
precision arithmetic where rounding errors become dominant near h~1e-7.

The 2×2 Hessian inversion is done analytically:
  [H11 H12]^(-1) = (1/det) · [H22 -H12; -H12 H11]
This is exact and avoids numerical inversion issues.

CIs in the original scale are obtained via the delta method / log-normal
approximation: θ_CI = [exp(log(θ) ± z·sd_log(θ))]. This is standard.

The code correctly validates that the determinant is positive (det > 1e-10)
and that the variances are positive before proceeding, with graceful failure.

Correlation of log-parameters is reported as: corr = cov / (sd_β · sd_η).
This is informative for practitioners (high |corr| indicates near-flat
likelihood surface, unstable estimates).

4.9 BOOTSTRAP CONFIDENCE INTERVALS
------------------------------------
Function: bootstrapWeibull()

VERDICT: CORRECT — well-implemented

Parametric bootstrap with B resamples (200, 500, or 1000, user-selectable).
For exact failures: samples drawn from W(β̂, η̂, γ̂) via quantile inversion.
For right-censored: censoring time fixed (Type I censoring assumption). Correct.
For interval-censored: latent failure time sampled from the conditional
distribution F(t | left < T < right), using the inverse CDF of the truncated
Weibull. This is the correct (and non-trivial) approach per Meeker & Escobar.

Percentile CI method: sort bootstrap estimates, take α/2 and 1-α/2 quantiles.
Standard and distribution-free. Correct.

Async chunked execution (20 resamples per chunk) with progress bar prevents
UI thread blocking. Good engineering.

B-lives (B10, B50) and MTTF are collected and reported from bootstrap.
Correct — these are functions of the fitted parameters.

NOTE: Bootstrap resamples that fail to converge are silently skipped
(try/catch). With the standard minimum-2-failure requirement, this could bias
the bootstrap distribution if many resamples fail. For well-conditioned data
this is unlikely, but for datasets near the identifiability boundary this
could be a concern. The code does not report the number of failed resamples.

MINOR ISSUE W-04: For 3P bootstrap, the resample is always fit as 2P
(weibullMLE with gam=0) unless use3param is true, in which case weibullMLE3
is called. However, weibullMLE3 is computationally expensive (200-point grid
+ golden section). For large B this is very slow. The code UI warns "slow"
but does not provide a faster inner-loop optimization for 3P bootstrap. This
is a performance issue, not a correctness issue.

4.10 BIAS CORRECTION (THOMAN/ROSS)
------------------------------------
Function: thomanBiasCorrection()

VERDICT: CORRECT — a previously noted fix was applied properly

Formula: β_corrected = β̂ · (n - 0.68) / n

This matches Ross (1994, eq. 3.8) and Thoman et al. (1969). The code comment
explicitly documents that a prior version incorrectly used (n-1)/(n-0.68)
instead, and that this was fixed (FIX F-02). The current formula is correct.

Conditions for applying:
  - MLE method only (not regression)
  - n < 30 (small sample)
  - Censoring fraction ≤ 20%

These conditions are reasonable. The Ross correction was derived for complete
samples; its applicability to censored data is debated. The code conservatively
skips the correction above 20% censoring with an explanatory toast notification.
This is appropriate scientific conservatism.

NOTE: Only β is bias-corrected; η is not adjusted accordingly. In principle,
if β is changed, η should be re-estimated for consistency. However, the bias
in η is substantially smaller than in β (Ross 1994), and this simplification
is standard in practice.

4.11 MTTF AND B-LIFE CALCULATIONS
-----------------------------------
Relevant code in calculateAndDisplay()

VERDICT: CORRECT

MTTF = γ + η · Γ(1 + 1/β)
B_x  = γ + η · (-ln(1 - x/100))^(1/β)

These are the exact Weibull mean and quantile function formulas. The Lanczos
gamma function (Section 4.13) is used for Γ(1 + 1/β). Correct.

B10 (= 10th percentile), B50 (median = η·(ln 2)^(1/β) when γ=0), and B90 are
all computed correctly. Note that B50 ≠ MTTF unless β ≈ 3.6 (log-normal-like).

4.12 LIKELIHOOD RATIO TEST (2P vs. 3P)
----------------------------------------
Function: lrTest2vs3()

VERDICT: PARTIALLY CORRECT — ISSUE FOUND

ISSUE W-05 (Moderate): The LR test statistic is:
  D = 2·(LL₃ - LL₂)

Under H₀: γ = 0, the code correctly notes (in a comment) that because γ is on
the boundary of the parameter space (γ ≥ 0), the asymptotic distribution of D
is NOT χ²(1) but rather a 50/50 mixture: 0.5·δ₀ + 0.5·χ²(1)
(Self & Liang 1987, Case 5).

The p-value formula in the code is:
  p = 1 - Φ(√D)

which is indeed 0.5·P(χ²(1) > D) = 0.5·(1 - Φ(√D)) × 2 = 1 - Φ(√D).
WAIT — let us verify: P(χ²(1) > D) = P(Z² > D) = 2·P(Z > √D) = 2·(1 - Φ(√D)).
Therefore 0.5·P(χ²(1) > D) = 1 - Φ(√D). So the formula IS correct for the
boundary-corrected test.

HOWEVER: The significance threshold is compared at p < 0.05, but this threshold
is already in the boundary-corrected p-value scale, so the comparison is
appropriate. The formula is in fact correct (the comment saying "p = 1 - Φ(√D)"
implements exactly the Self-Liang boundary correction as described).

REVISED VERDICT: The LR test is CORRECT. The boundary correction is properly
applied via the formula p = 1 - Φ(√D), consistent with Self & Liang (1987).

4.13 GAMMA FUNCTION IMPLEMENTATION
------------------------------------
Function: gammaFn()

VERDICT: CORRECT

Lanczos approximation with g=7 and 9 coefficients. This is the Spouge/Lanczos
implementation widely used in numerical software. Relative error < 10^-13 for
Re(z) > 0.5 (Numerical Recipes, Press et al. 2007). Correct for all Weibull
shape parameters of practical interest (β ∈ (0, 100]).

The reflection formula for z < 0.5 is correctly implemented:
  Γ(z) = π / (sin(πz) · Γ(1-z))

4.14 NORMAL CDF APPROXIMATION
-------------------------------
Function: normalCDF()

VERDICT: CORRECT — known precision

Abramowitz & Stegun (26.2.17) rational approximation. Maximum absolute error
≈ 7.5×10^-8. This is sufficient for the p-value comparisons done in this tool.

Fixed z-scores for confidence intervals (90%: 1.645, 95%: 1.96, 99%: 2.576)

4.15 OUTLIER DETECTION
------------------------
Function: detectOutliers()

VERDICT: ACCEPTABLE with note

Method: Tukey IQR fences on log-transformed data (log scale is appropriate for
right-skewed reliability data). Fence: [Q1 - 1.5·IQR, Q3 + 1.5·IQR].

NOTE: Minimum sample size of 4 before any detection attempt is appropriate.
NOTE: Quantile computation uses Math.floor() index, which is a crude percentile
interpolation. For small samples (n=4 to 8) this can give imprecise quartile
estimates. The tool uses outlier detection only for advisory warnings (toast),
not for parameter estimation, so imprecision here is acceptable.

4.16 MIXTURE DETECTION (CURVATURE TEST)
-----------------------------------------
Function: detectMixture()

VERDICT: CORRECT with note

Uses Spearman rank correlation between x-values (ln(t)) and |residuals| from
the Weibull probability plot. A significant positive correlation indicates
systematic curvature (residuals larger for extreme times). This is a valid
diagnostic for non-Weibull behavior.

The critical ρ is derived from the t-approximation for Spearman's ρ:
  t = ρ·√(n-2) / √(1-ρ²)
with appropriate critical t-values tabulated for df = n-2.

The table used (t_crit: 3.182 for df=4, 2.571 for df=5, ...) matches standard
Student's t tables at α=0.05 two-tailed. Correct.

Tie handling uses average ranks (Zar 1984). Correct.

5.1 INPUT PARSING AND NUMBER LOCALIZATION
------------------------------------------
Function: parseNumber(), parseInput()

VERDICT: GOOD — well thought out

The parser handles:
  US decimal:      1234.56
  EU decimal:      1234,56
  EU thousands:    1.234.567
  US thousands:    1,234,567
  US+decimal:      1,234.56
  EU+decimal:      1.234,56
  FR space-sep:    1 234,56
  Ambiguous:       1,234 (flagged, default EU interpretation, user warned)

The ambiguity flag for "1,234" (annotated as FIX F-01) is a professional
touch — silently coercing this could introduce large errors (factor 1000)
in failure times. The warning toast is appropriately prominent.

Delimiter handling: split on /[\n;,]+/ — note this consumes the commas used
as decimal separators in EU notation. This is correctly handled by parseNumber()
operating on individual tokens rather than the full string. The architecture
is sound.

NOTE: Interval pairs are parsed by a separate regex expecting format "start-end".
Negative numbers in intervals are not supported (not relevant for time data).
Scientific notation is supported in intervals via the regex
/^(\d+(?:\.\d+)?(?:[eE][+-]?\d+)?)/ but NOT in the main failure time parser
(parseNumber does not handle scientific notation). This inconsistency could
cause silent parsing failures for very large/small times in scientific notation.

ISSUE W-06 (Minor): Scientific notation (e.g., "1.5e3") is not handled by
parseNumber(). The function uses parseFloat() internally, which does parse
scientific notation, but the initial split on /[\n;,]+/ could split "1.5e-3"
incorrectly at the comma, producing "1.5e-3" as a single token (actually fine,
since e notation uses no commas). Testing reveals this works by accident for
"e" notation, but not for tokens like "1,5e3" where the EU decimal comma
conflicts with the delimiter.

5.2 EDGE CASE AND DATA VALIDATION
-----------------------------------
VERDICT: GOOD

- Minimum failures enforced: 2 for 2P, 3 for 3P (correct).
- Degenerate data (CV < 0.001 = all identical) caught before MLE.
- Negative failure times rejected (filter v > 0 in parseInput).
- n=2 case: GoF tests suppressed with explanation (correct — KS needs n≥3).
- Convergence failure: Nelder-Mead non-convergence is flagged with a warning.
- Censoring > 50%: CI warning displayed (correct — high censoring inflates CI).
- β > 50: plotting clamped but reported value unaffected (correct).
- γ near min(t): warning displayed (Section 4.3).
- Interval left ≥ right: pair silently rejected (parseIntervalPairs filters
  right > left). A warning to the user about rejected intervals would be better.
- All-suspended data: no exact failures → alert raised. Correct.

5.3 NUMERICAL STABILITY
------------------------
VERDICT: GOOD

Key safeguards:
- Log-space Hessian for Fisher CI (avoids scale mismatch).
- R(L) - R(R) for interval likelihood (avoids cancellation near F=1).
- Number.EPSILON as floor for interval probability (avoids log(0)).
- 1e-12 floor for exact failure CDF (avoids log(0) in AD statistic).
- 1e-15 ceiling for (1 - CDF) in AD statistic.
- Minimum β bound: 1e-4 in negLogLik, 0.05 in returned parameters.
  NOTE: The effective lower bound is 0.05 (from max(0.05, res.x[0])) rather
  than the tighter 1e-4 used in the objective function. This inconsistency
  means the optimizer can temporarily explore β < 0.05 during optimization
  (legitimate) but cannot return β < 0.05 as a final answer (capped). This
  is arguably acceptable since β < 0.05 is unrealistic for physical systems.

- Underflow in exp(-x) for large x: not explicitly guarded. For very large
  (t/η)^β values (e.g., t=10^6, η=1, β=5), the exponential argument is
  10^30, causing Math.exp() to return 0 (JavaScript behavior). The result
  is F(t) = 1 and R(t) = 0, which is correct in the limit, so this is
  acceptable.

5.4 OPTIMIZATION ENGINE (NELDER-MEAD)
---------------------------------------
Function: nelderMead()

VERDICT: CORRECT — standard implementation

Standard Nelder-Mead simplex method (Nelder & Mead 1965) with:
  α=1.0 (reflection), γ=2.0 (expansion), ρ=0.5 (contraction),
  σ=0.5 (shrinkage). These are the canonical parameter values.

maxIter=800 is reasonable for a 2D problem.
Convergence criterion: max spread in function values AND parameter values < tol.
Dual criterion (function + parameter space) is more robust than single-criterion.

POTENTIAL ISSUE: Perturbation for zero-valued starting parameters:
  perturbStandard(): if x[i]=0 → use 0.00025
Starting β=0 or η=0 is prevented by the initial bounds (β₀ from Dubey ≥ 0.1,
η₀ ≥ 0.1), so this branch should never be exercised in practice.

7. COMPARISON WITH REFERENCE COMMERCIAL SOFTWARE

The following comparisons are based on documented expected behaviors of
commercial tools and the R statistical ecosystem (WeibullR, fitdistrplus,

7.1 WEIBULL++ (ReliaSoft)
--------------------------
Weibull++ is the industry-standard commercial Weibull tool.

  MLE parameters (complete data): Expected agreement within 0.01% for n ≥ 5.
    Tool uses same log-likelihood formulation. EXPECTED TO AGREE.

  Censored data MLE: ReliaSoft uses L-BFGS-B or Marquardt; this tool uses
    Nelder-Mead. Both should converge to the same optimum for well-conditioned
    data. Slight differences possible for degenerate or highly censored cases.

  Confidence intervals: ReliaSoft reports Likelihood Ratio (LR), Fisher Matrix,
    and Beta-Binomial CIs. This tool provides LR and Fisher Matrix. Agreements
    with LR CIs expected for typical cases; deviations possible near boundaries.

  KS/AD tests: ReliaSoft applies the Stephens (1977) correction for estimated
    parameters. This tool applies the same AD* = AD·(1 + 0.2/√n) correction.
    Expected agreement for A² ≥ 0.2; divergence for A² < 0.2 (Issue W-02).

  3P estimation: ReliaSoft uses a different numerical optimizer (gradient-based).
    This tool uses grid + golden section. Results should agree within
    optimization tolerance for n ≥ 6; may differ for small n (n=3,4).

  Rank methods: ReliaSoft supports Median Rank and Benard's. This tool supports
    both (Hazen = "median rank" using (j-0.5)/N, Benard's = (j-0.3)/(N+0.4)).
    NOTE: There is a naming inconsistency: the dropdown labels Hazen as
    "Hazen — F=(j−0.5)/N" which is correct, but the Weibull++ default "median
    rank" typically refers to Benard's approximation, not Hazen. Users migrating
    from Weibull++ should use "Benard" to match defaults.

7.2 MINITAB (Reliability/Survival Analysis)
--------------------------------------------
Minitab uses MLE with Anderson-Darling as the primary GoF test. It reports
both Fisher-matrix and likelihood-ratio CIs.

  Parameters: Expected agreement within numerical tolerance.
  GoF: Minitab uses Weibull-specific tabulated critical values for AD
    (not the generic Marsaglia series). Results may differ slightly from
    this tool's AD p-values, especially for A² < 0.2 (Issue W-02).
  Censored data: Minitab supports all censoring types. Expected agreement.

7.3 R: WeibullR PACKAGE
-------------------------
WeibullR implements MLE and regression for 2P and 3P Weibull.

  MLE (2P): WeibullR uses optim() with L-BFGS-B. Expected parameter agreement.
  LR CIs: WeibullR provides profile likelihood CIs. Expected close agreement.
  GoF: WeibullR uses Filliben correlation coefficient (different from KS/AD).
  Bootstrap: WeibullR provides parametric and non-parametric bootstrap.
    This tool's parametric bootstrap matches WeibullR's parametric approach.

7.4 R: fitdistrplus PACKAGE
-----------------------------
fitdistrplus uses maximum likelihood for standard distributions.

  For 2P Weibull MLE on complete data: identical formulation, expected agreement.
  For censored data: fitdistrplus supports interval and right-censoring.
    Expected parameter agreement.
  CIs: fitdistrplus uses asymptotic normal CIs on log scale (same as Fisher
    matrix here). Expected agreement.

7.5 EXCEL/SPREADSHEET TOOLS
-----------------------------
Many practitioners use Excel-based Weibull tools with regression (probability
plot). This tool's regression implementation matches Excel-based approaches
(Benard/Hazen median ranks + OLS in ln-ln space). Agreement expected.

8. ISSUES REGISTRY: CONFIRMED BUGS AND DEFICIENCIES
================================================================================

SEVERITY SCALE: CRITICAL | HIGH | MODERATE | MINOR | INFO

ID      SEVERITY    DESCRIPTION
----------------------------------------------------------------------
W-01    MINOR       3P golden-section refinement window may truncate the
                    search interval when bestGamma is within 1 grid step
                    of the boundary [0, 0.9999·min(t)]. Could miss ~0.5%
                    of search space near the boundary. Impact: negligible
                    in practice.

W-02    MODERATE    Anderson-Darling p-value for A² < 0.2 is computed
                    using a series that underflows to 0 for small A², 
                    producing p ≈ 0 (extremely poor fit) when the correct
                    value should be p → 1 (excellent fit). The GoF traffic
                    light would incorrectly show RED for a perfect-fitting
                    dataset. Real-world impact: limited, as A² < 0.2 is
                    uncommon for typical reliability sample sizes.

W-03    MINOR       Gamma CI bisection for 3P mode searches with 40
                    iterations (vs. 50 for β and η CIs). Precision is
                    still ~2^(-40) ≈ 10^(-12), adequate. Consistency
                    improvement possible.

W-04    INFO        Bootstrap for 3P Weibull calls the full weibullMLE3()
                    (200-point grid + golden section) at each resample.
                    Performance degrades significantly for B=1000 and 3P.
                    This is a performance concern, not correctness. A
                    warm-started 2-step optimizer could reduce this cost
                    by ~10×.

W-05    RESOLVED    LR 2P/3P test: initial concern about boundary correction.
                    After detailed review, the formula p = 1 - Φ(√D) is
                    correct for the Self-Liang boundary mixture distribution.
                    (See Section 4.12.)

W-06    MINOR       Scientific notation (e.g., "1.5e3") not explicitly
                    handled in parseNumber(). Works by accident for standard
                    "e" notation in isolation but may fail for EU decimal
                    notation combined with exponents (e.g., "1,5e3").

9. ISSUES REGISTRY: WARNINGS AND DESIGN CONCERNS
================================================================================

ID      CONCERN     DESCRIPTION
----------------------------------------------------------------------
D-01    DESIGN      Regression CIs use the heuristic formula:
                      β_CI ≈ β ± z·β/√n
                    This is not a statistically derived formula; it assumes
                    the relative standard deviation of β̂ is 1/√n, which
                    is generally not true for regression estimators. The
                    tool labels these as "heuristic — not statistically
                    derived" and suppresses them for n<10, but they
                    remain potentially misleading for n ∈ [10, 20].
                    Recommendation: suppress regression CIs entirely or
                    replace with bootstrap-based intervals.

D-02    DESIGN      The Hazen median rank (j-0.5)/N is labeled as the
                    default in the dropdown. Benard's approximation is
                    generally preferred in reliability engineering (closer
                    to true median rank). Default should arguably be
                    Benard's to match Weibull++ and Abernethy conventions.

D-03    DESIGN      Outlier detection is performed on exact failure times
                    only, using the raw IQR method on log-scale. Suspen-
                    sions are excluded from outlier detection. Intervals
                    are also excluded. A practitioner might want to be
                    warned about unusual interval boundaries.

D-04    DESIGN      The bias correction (Thoman/Ross) is applied to the
                    MLE β̂ but not propagated consistently to B-lives and
                    MTTF in the CI computation. CIs are computed using
                    the uncorrected β, but the displayed B-lives use the
                    corrected β. This could create slight inconsistency
                    between reported CIs and central estimates.

D-05    DESIGN      The GoF traffic light threshold for "green" (good fit)
                    is shifted from p>0.05 to p>0.15 to compensate for
                    anticonservative bias. This correction is ad hoc. The
                    scientifically rigorous approach would be to use
                    Lilliefors-corrected critical values specific to the
                    Weibull distribution (Stephens 1977 tables).

D-06    DESIGN      The KS statistic is computed only on exact failure
                    times, ignoring the censoring pattern. The Kaplan-Meier
                    based analog (Koziol-Green test or the Kim & Schroeder
                    statistic) would be more appropriate for censored
                    samples. The current tool correctly warns about this
                    limitation but still reports the number.

D-07    UI          The probability plot x-axis label reads "ln(t-γ)"
                    which is correct for 3P mode but slightly confusing
                    for 2P where γ=0 (label should read "ln(t)").

D-08    INFO        Bootstrap resamples that fail MLE fitting are silently
                    discarded. The final bootstrap sample size N_success
                    is not reported to the user. If many resamples fail
                    (near-degenerate data), the CI width could be under-
                    estimated due to selection bias toward stable samples.

10. RECOMMENDATIONS
================================================================================

PRIORITY 1 (Fix recommended):
  W-02: Replace the A² < 0.2 branch of adPvalueMarsaglia() with the correct
        asymptotic tail expansion. Suggested formula:
          p ≈ (√(2π)/A²) · Σ_{j=0}^{K} (-1)^j·(4j+1)·exp(-(4j+1)²π²/(8A²))
        evaluated in higher precision, or use the complementary expansion
        that avoids the large-c underflow.

  D-04: When bias correction is applied to β, recompute B-lives and MTTF
        and then use corrected parameters as input to CI calculation as well,
        or clearly document that CIs are always based on raw MLE β̂.

PRIORITY 2 (Improvement recommended):
  D-01: Remove the heuristic regression CIs or clearly flag them as
        "illustrative only." Provide bootstrap CIs as the alternative for
        regression method users.

  D-08: Report the number of successful bootstrap resamples (N_success / B)
        in the bootstrap CI output panel.

  W-06: Add explicit scientific notation support in parseNumber().

PRIORITY 3 (Nice to have):
  D-02: Change the default rank method to Benard's for better consistency
        with Weibull++ and standard reliability engineering practice.

  D-05: Add optional Stephens (1977) table-based critical values for the
        Weibull AD test, replacing the ad hoc 0.15 threshold.

  W-04: Implement a warm-started 2-step optimizer for 3P bootstrap to reduce
        computation time by approximately 10×.

  D-07: Update probability plot axis label to "ln(t)" when γ=0.

  General: Add a formal test suite (unit tests) for critical math functions,
        covering at minimum: gamma function, normal CDF, KS/AD p-values,
        MLE parameter recovery for known Weibull distributions.
