1. EXECUTIVE SUMMARY
--------------------
The audited software is a browser-based, single-file HTML/JavaScript Weibull
reliability analysis tool implementing 2-parameter and 3-parameter Weibull
distributions with Maximum Likelihood Estimation (MLE), Linear Regression
(probability-plot), parametric bootstrap confidence intervals, profile
likelihood and Fisher Information Matrix intervals, goodness-of-fit tests
(Kolmogorov-Smirnov and Anderson-Darling), and several secondary analyses
(mixture detection, outlier detection, bias correction, LR test 2P vs 3P,
warranty KPI).

Overall quality assessment: GOOD to VERY GOOD.

The mathematical foundations are correct and internally consistent. The
algorithmic implementations are above average for a single-file browser tool
and compare favourably with open-source reference implementations. Several
advanced and professionally appropriate choices have been made (profile
likelihood CIs, boundary-corrected LR test, log-parametric Hessian,
Herd-Johnson adjusted ranks). A small number of issues have been identified
ranging from minor inaccuracies to moderate methodological limitations.

Critical issues: 0
Major issues:    3
Minor issues:    7
Observations:    9

3. MATHEMATICAL AND STATISTICAL AUDIT
--------------------------------------

3.1  Core Weibull Functions (PDF, CDF, Hazard, Reliability)
------------------------------------------------------------
Implementations reviewed (lines ~1762-1781):

  PDF:  f(t) = (β/η) · ((t-γ)/η)^(β-1) · exp(-((t-γ)/η)^β)
  CDF:  F(t) = 1 - exp(-((t-γ)/η)^β)
  h(t) = (β/η) · ((t-γ)/η)^(β-1)
  R(t) = exp(-((t-γ)/η)^β)

VERDICT: CORRECT. All four functions are textbook Weibull with location
parameter γ. Boundary conditions (t ≤ γ) correctly return 0, 0, 0, and 1
respectively. Consistent with ISO 3977-9, Nelson (1982), Meeker & Escobar
(1998).

The hazard function uses the reduced formula h(t) = f(t)/R(t) = (β/η)·((t-γ)/η)^(β-1),
which is exact and avoids catastrophic cancellation. No issues found.


3.2  Gamma Function Implementation
-----------------------------------
Implementation (lines ~1628-1637): Lanczos approximation with g=7 and the
9-coefficient Lanczos table from Numerical Recipes (Press et al. 2007).

  Coefficients match the standard Lanczos g=7 table (Spouge 1994 / NR).
  Reflection formula applied correctly for z < 0.5.
  Accuracy: typically ~15 significant digits across the positive real axis.
  This exceeds IEEE 754 double precision requirements.

VERDICT: CORRECT. The implementation is standard and appropriate. The gammaFn
is used exclusively for MTTF calculation: MTTF = γ + η·Γ(1+1/β), which is
mathematically correct.


3.3  MTTF and B-Life Percentiles
----------------------------------
MTTF = γ + η·Γ(1 + 1/β)                         [correct]
B_p  = γ + η·(-ln(1-p))^(1/β)                    [correct]

  B10 uses p=0.10: γ + η·(-ln(0.9))^(1/β)        [correct]
  B50 uses p=0.50: γ + η·(-ln(0.5))^(1/β)        [correct]
  B90 uses p=0.90: γ + η·(-ln(0.1))^(1/β)        [correct]

All formulas are the standard Weibull quantile function. VERDICT: CORRECT.

NOTE: When bias correction is applied (betaUsed ≠ beta), MTTF and B-lives
are recalculated with corrected β. This is appropriate.

MINOR ISSUE M1: The variable eta is NOT bias-corrected (etaCorr = etaUsed = eta
without change), while β is corrected. For MLE Weibull, correcting only β and
not recalibrating η introduces a small inconsistency: η is implicitly the MLE
scale at the uncorrected β. The correct approach for the Thoman/Ross correction
is to keep η as estimated (the MLE estimate of η is approximately unbiased),
so this is acceptable in practice. However, the UI label "uncorrected" next to
η could mislead users. A clarifying tooltip would improve transparency.


3.4  Median Rank / Adjusted Rank Estimation (Herd-Johnson)
-----------------------------------------------------------
Implementation (lines ~1787-1816):

The code implements Herd-Johnson adjusted ranks for right-censored data,
following Abernethy (2006) p.3-3 exactly. The increment formula is:

  Δj_i = (N+1 - j_{i-1}) / (N+1 - i)

where i is 0-based in the code but the denominator correctly accounts for
1-based positioning: N+1-(i+1)+1 = N+1-i. This is verified against Abernethy's
tabulated examples.

Tie handling: when failure time equals suspension time, failures are placed
first (Abernethy 2006). This is correct and explicitly commented.

Median rank approximations offered:
  - Benard: F = (j - 0.3) / (N + 0.4)   [standard, correct]
  - Hazen:  F = (j - 0.5) / N            [standard, correct]

Clamping to [0.0001, 0.9999] avoids log(-log(0)) = ±∞. VERDICT: CORRECT.


3.5  Linear Regression (Probability Plot) Method
-------------------------------------------------
The Weibull probability plot linearisation is:

  Y = ln(-ln(1-F)) = β·ln(t-γ) - β·ln(η)

so slope = β, intercept = -β·ln(η), giving η = exp(-intercept/β).

Implementation (lines ~1818-1831): standard OLS in the linearised space.
VERDICT: CORRECT.

MINOR ISSUE M2: The regression confidence intervals for β and η (lines 2659-
2660) use a heuristic formula: β ± z·β/√n. This is NOT a statistically valid
CI for the regression slope. The correct approach would be to use the standard
error of the OLS slope in the Weibull plot. The code appropriately suppresses
these CIs for n<10 and labels them "(approx, n≥10)", but the formula itself
is an ad-hoc approximation not traceable to a statistical reference. Users
should be warned more prominently that regression CIs are heuristic.

3-parameter regression (CCO method, lines 2097-2121): performs a grid search
over γ to maximise R² in the probability plot. This is the classical Confidence
of Fit / Correlation Coefficient Optimization (CCO) approach. Grid has 400
points over [0, 0.9999·min(t)]. This is correct in principle.

MINOR ISSUE M3: For 3P regression, no golden-section refinement is applied
after the coarse grid (unlike 3P MLE which uses golden-section). The 400-point
grid may miss the true optimum with resolution of only ~0.25% of min(t) per
step. This is a minor accuracy limitation; the MLE path has no such deficiency.


3.6  Maximum Likelihood Estimation — 2-Parameter
-------------------------------------------------
Log-likelihood for exact observations:
  LL = ln(β) - β·ln(η) + (β-1)·ln(t) - (t/η)^β           [correct]

Right-censored contribution:
  LL = -(t/η)^β  [i.e., ln(R(t))]                          [correct]

Interval-censored contribution:
  LL = ln(R(left) - R(right))                               [correct]
  Using: prob = exp(-(left/η)^β) - exp(-(right/η)^β)

Numerical fix for interval censoring (lines 1911-1916): uses R(left)-R(right)
to avoid catastrophic cancellation when F→1, and floors to Number.EPSILON
instead of an arbitrary small constant. This is the professionally correct
approach (Meeker & Escobar 1998, §5.3).

Initial values (lines 1934-1940): Dubey (1967) estimator β₀ ≈ 1.283/σ_ln.
This is a recognised and more stable starting point than CV-based heuristics.
VERDICT: CORRECT.

Multi-start strategy (4 starting points including β₀·0.5, β₀·2, and β=1):
appropriate for avoiding local minima in flat or multimodal likelihood surfaces.
VERDICT: CORRECT.

MINOR ISSUE M4: The constraint `if (beta <= 0.01 || eta <= 0.01) return 1e12;`
(line 1894) uses a fixed lower bound of 0.01 for both parameters. For failure
datasets with very small time values (e.g., milliseconds) or very large β
(sharp wearout), this bound could occasionally be hit during optimizer
exploration. A tighter lower bound of 1e-4 would be more robust. This is
cosmetic in most practical scenarios.


3.7  Maximum Likelihood Estimation — 3-Parameter
-------------------------------------------------
Algorithm (lines ~2010-2085):
  Step 1: Coarse grid — 200 points for γ in [0, 0.9999·min(t)]
           For each γ, fit 2P MLE with useMultiStart=false (speed)
           Select γ with maximum log-likelihood
  Step 2: Golden-section refinement ±1 grid step around best γ (25 iterations)
  Step 3: Final 2P MLE with useMultiStart=true for quality

Golden-section ratio: (√5-1)/2 ≈ 0.618. Correctly implemented (lines 2042-48).
Convergence tolerance: 1e-8 in γ space.

VERDICT: CORRECT in structure. This follows the standard iterative approach
for 3P Weibull MLE (Lawless 1982; Nelson 1982).

MINOR ISSUE M5: The upper bound for γ is strictly 0.9999·min(t). This means
γ can never equal or exceed min(t), which is correct (the location parameter
must be strictly less than all observed times). However, for degenerate cases
where multiple observations share the exact minimum value, the effective search
range may be very narrow, potentially causing the golden-section to converge
to a numerically uncertain γ. A warning should be displayed in this case.

OBSERVATION O1: For 3P MLE, the likelihood surface for γ is known to be
unbounded as γ → min(t⁻) when β < 1 (Cheng & Amin 1983). The code does not
explicitly guard against or warn about this pathological case. Reliable
commercial tools (ReliaSoft, Minitab) handle this by either bounding the
search or issuing a dedicated warning. This is a known theoretical limitation,
not a coding error, but deserves explicit user communication.


3.8  Nelder-Mead Optimizer
---------------------------
Implementation (lines ~1838-1888): Standard Nelder-Mead simplex with:
  α=1.0 (reflection), γ_nm=2.0 (expansion), ρ=0.5 (contraction),
  σ=0.5 (shrink). These are the Nelder & Mead (1965) original parameters.

Convergence criterion (line 1885): max absolute difference in both function
values and parameter values < tol (1e-8). This is a sound dual criterion.

Termination: maxIter=800, which is generous for a 2-parameter problem.

Perturbation for initial simplex (lines 1833-1836): val + 5% of val (or 0.00025
for zero). This is reasonable.

VERDICT: CORRECT. Standard and well-implemented.

OBSERVATION O2: Nelder-Mead is not guaranteed to find the global optimum.
For 2P Weibull MLE with complete data, the likelihood is strictly unimodal
and gradient-based methods would be faster and guaranteed convergent. The
choice of Nelder-Mead is appropriate for generality (censored/interval data,
3P), but a comment explaining this choice would improve maintainability.


3.9  Profile Likelihood Confidence Intervals
--------------------------------------------
Implementation (lines ~2136-2234):

For each parameter, the profile likelihood is computed by fixing that parameter
at a test value and re-optimising the other parameter. The CI boundary is
found where the negative log-likelihood exceeds:

  LL_min + χ²_{1,α}/2

χ² critical values used:
  90%: 2.706, 95%: 3.841, 99%: 6.635   [all correct per χ²(1) distribution]

Boundary search: exponential expansion + bisection (50 iterations), achieving
2^(-50) ≈ 1e-15 relative precision. VERDICT: CORRECT and robust.

OBSERVATION O3: The profile likelihood is computed marginally for each
parameter separately (marginal profile, not conditional profile). This is the
standard approach and correct for reporting individual parameter CIs. Joint
confidence regions are not offered, which is appropriate for a tool of this
scope.

OBSERVATION O4: For γ CI in 3P models, the code re-runs the full 2P MLE for
each γ evaluation (lines 2192-2197). This is computationally expensive but
statistically correct. No accuracy issue identified.


3.10 Fisher Information Matrix (Asymptotic) CIs
------------------------------------------------
Implementation (lines ~2255-2312):

The Hessian is computed numerically in LOG-PARAMETER space (log β, log η) with
central finite differences at step h=1e-4. This is a superior approach compared
to differentiating in native parameter space, as the log-normal approximation
is more accurate and the parameters are naturally positive.

Hessian inversion (2×2 analytic):
  det = H11·H22 - H12²
  var(log β) = H22/det, var(log η) = H11/det

CI on original scale via log-normal approximation:
  β_low/high = β·exp(∓ z·√var(log β))

VERDICT: MATHEMATICALLY CORRECT. This is the standard asymptotic CI for
MLE, implemented with good numerical hygiene (log space).

MINOR ISSUE M6: The finite difference step h=1e-4 is fixed regardless of
parameter magnitudes. For very large η (e.g., η=1e7 for years in hours),
the step in log space (h=1e-4) is fine, but for very small η, precision
may degrade. An adaptive step (h proportional to parameter magnitude) would
be more robust. In practice this rarely causes problems.


3.11 Parametric Bootstrap CIs
-------------------------------
Implementation (lines ~2451-2553):

Generates B resamples (default 500) from the fitted Weibull distribution using
the quantile function (exact inverse CDF). Right-censored observations are kept
at fixed censoring times (Type I censoring). Interval-censored observations
retain original interval boundaries.

MAJOR ISSUE MAJ1: The bootstrap resampling for interval-censored data (lines
2487-2490) keeps the exact original intervals rather than generating new
observations within each interval. This means interval-censored bootstrap
resamples are deterministic (no variation from censored intervals), leading
to underestimated bootstrap variance when the proportion of interval-censored
data is high. For mixed datasets with significant interval censoring, bootstrap
CIs will be too narrow. The correct approach is to sample the latent failure
time uniformly (or from the truncated Weibull) within each interval.

For 3P bootstrap (lines 2494-2503): the code uses 2P MLE shifted by the
estimated γ as an approximation rather than re-running 3P optimisation for
each resample. This is a deliberate speed tradeoff (documented in a comment),
which is acceptable but introduces bias in the γ distribution. Bootstrap CIs
for 3P β and η will be slightly wider than true, and γ CI is not reported
from bootstrap.

Percentile CI method (linear interpolation, lines 2530-2534): correct.

OBSERVATION O5: The bootstrap uses Math.random() (Xorshift128+, not
cryptographically secure, but adequate for statistical simulation). No
seeding is provided, making results non-reproducible between runs. A seed
option would improve scientific reproducibility, though this is a feature
request rather than a defect.


3.12 Small-Sample Bias Correction (Thoman / Ross)
--------------------------------------------------
Implementation (lines ~2238-2243):

  β_corrected = β_MLE · (n-1) / (n-0.68)

The code attributes this to Ross (1994) and calls it an approximation of
the Thoman et al. (1969) correction. Let us examine this carefully.

The Thoman et al. (1969) bias correction is based on Monte Carlo-derived
correction factors c(n) for complete samples:
  β_unbiased ≈ β_MLE · c(n)

The Ross (1994) analytical approximation is:
  c(n) ≈ (n-1)/(n-0.68)      [for complete data]

This is approximately correct for n=5..30 and complete datasets. Relative
error versus tabulated Thoman factors: typically <2% for n≥5.

The code correctly restricts bias correction to:
  - MLE method only (not regression)
  - n < 30
  - Censoring fraction ≤ 20%

VERDICT: APPROPRIATE. The restriction to ≤20% censoring is conservative and
correct since the bias correction is derived for complete data.

MINOR ISSUE M7: The code label says "Thoman et al. 1969" in the UI but
implements the Ross (1994) formula. This is technically an approximation of
Thoman's result, not the exact tabulated factors. For strict correctness, the
label should read "Ross (1994) approximation to Thoman et al. (1969)" or
include the full correction table for n=2..30.


3.13 Goodness-of-Fit: Kolmogorov-Smirnov Test
----------------------------------------------
Statistic D (lines ~1970-1975): D = max over all i of |F_i^emp - F_i^theo|,
where both the empirical CDF upper and lower step values are compared.
VERDICT: CORRECT (two-sided KS statistic).

P-value computation (lines 1639-1648): Kolmogorov (1933) asymptotic series:
  P(D > d) ≈ 2·Σ_{k=1}^{∞} (-1)^{k+1} · exp(-2k²λ²)

where λ = (√n + 0.12 + 0.11/√n)·D (Stephens 1974 continuity correction).

VERDICT: CORRECT. The Stephens correction improves accuracy for finite n.
Series truncated at 60 terms with 1e-12 tolerance: adequate.

MAJOR ISSUE MAJ2 (STATISTICAL): The KS test is valid for KNOWN (fixed)
parameters. When parameters are estimated from the same data (as is always
the case here), the critical values are anti-conservative: the true null
distribution is shifted toward smaller D values. The correct test would be
the Lilliefors test for the Weibull distribution. The code acknowledges this
limitation explicitly with a warning in the GoF section, but the computed
p-values are still displayed numerically, which may be over-interpreted by
non-expert users.

Reference: Lilliefors (1969); Stephens (1974, 1977).
This issue is shared by virtually all commercial tools (ReliaSoft, Minitab)
in their default KS output; the warning is appropriate mitigation.


3.14 Goodness-of-Fit: Anderson-Darling Test
--------------------------------------------
Anderson-Darling statistic (lines ~1991-1993):
  A² = -n - (1/n)·Σ_{k=1}^{n} (2k-1)·[ln(F_k) + ln(1-F_{n+1-k})]

VERDICT: CORRECT. This is the standard formula (Anderson & Darling 1952).

Modification factor (line 1993): A²* = A²·(1 + 0.2/√n) — Stephens (1974)
small-sample correction for the AD statistic. CORRECT.

P-value via Marsaglia & Marsaglia (2004) series (lines 1651-1681): The code
implements both a standard branch (A² ≥ 0.2) and a small-A² branch with 500
terms. The Marsaglia (2004) series converges slowly for small A², and the
code correctly increases the number of terms in that regime.

VERDICT: CORRECT. Same anti-conservative caveat as KS applies (see MAJ2
above). The code's warning is adequate.

OBSERVATION O6: The Anderson-Darling test is generally more powerful than
KS for detecting deviations in the tails, which is particularly important
for reliability analysis. The code correctly prioritises AD in its GoF
traffic light (lines 2693-2707). This is the appropriate choice.


3.15 Likelihood Ratio Test 2P vs 3P
-------------------------------------
Implementation (lines ~2314-2333):
  D = 2·(LL₃ - LL₂)
  p = 1 - Φ(√D)   [boundary-corrected, Self & Liang 1987]

The boundary correction is critical here: since γ ≥ 0 (boundary of the
parameter space), the standard χ²(1) asymptotic distribution for the LR
statistic is incorrect. Self & Liang (1987) show the correct distribution
is 0.5·δ₀ + 0.5·χ²(1), giving p = 0.5·P(χ²(1) > D) = 1 - Φ(√D).

VERDICT: CORRECT. This is more statistically rigorous than most commercial
tools which use the naïve χ²(1) p-value.

The code handles D ≤ 0 correctly by returning p=1.0 (no improvement).


3.16 Mixture / Multi-Mode Detection (Spearman Curvature)
---------------------------------------------------------
Implementation (lines ~2370-2401):

Computes Spearman rank correlation between x-values (ln(t)) and absolute
residuals from the fitted line in the Weibull probability plot. High |ρ|
suggests curvature (non-Weibull or mixed populations).

Critical ρ threshold: n-dependent, using t-approximation:
  t_crit / √(t_crit² + df)   [Zar 1984 formula]

with df-specific critical t: df≥20 → 2.086, df≥10 → 2.228, df≥5 → 2.776,
else 3.182. These correspond approximately to α=0.05 two-tailed.

VERDICT: REASONABLE but heuristic. This is not a standard published test for
Weibull mixture detection; it is an approximate curvature test. Results should
be treated as exploratory. Average-rank tie handling is correctly implemented
(lines 2376-2388, Zar 1984).


3.17 Outlier Detection
-----------------------
Implementation (lines ~2351-2367):

IQR method applied in LOG space: Q1 and Q3 of ln(t), flag values where
ln(t) < Q1 - 1.5·IQR or ln(t) > Q3 + 1.5·IQR.

VERDICT: APPROPRIATE. Outlier detection in log space is correct for
right-skewed failure time data (Weibull, lognormal). The standard IQR fence
is conventional (Tukey 1977).

OBSERVATION O7: Q1 and Q3 are estimated using floor-based indexing:
  Q1 = sorted[floor(n·0.25)], Q3 = sorted[floor(n·0.75)]
This is the Type 1 quantile method. More accurate would be linear
interpolation (R's default type 7). For small n this can misplace Q1/Q3
by one rank. Impact: negligible in practice.


3.18 Normal CDF Approximation
-------------------------------
Implementation (lines ~2246-2252): Abramowitz & Stegun (1964) §26.2.17,
rational approximation with coefficients:

  a1=0.319381530, a2=-0.356563782, a3=1.781477937,
  a4=-1.821255978, a5=1.330274429
  p=0.2316419

This is the standard 5-term polynomial approximation with max error ~7.5e-8.
Used only in the LR test 2P vs 3P (normalCDF(√D)), so this precision is
more than adequate.

VERDICT: CORRECT.


4. NUMERICAL AND COMPUTATIONAL AUDIT
--------------------------------------

4.1 IEEE 754 Double Precision Handling
Interval censoring (prob = R_left - R_right): uses Number.EPSILON as a floor
instead of an arbitrary constant. This maximises floating-point resolution.
GOOD practice.

4.2 Overflow/Underflow Guards
The log-likelihood function returns 1e12 for invalid parameter ranges
(β ≤ 0.01, η ≤ 0.01, t-γ ≤ 0). This prevents NaN propagation into the
optimizer. ADEQUATE.

For β > 50, chart plotting is clamped with a warning (line 2428). This
prevents numerical overflow in (t/η)^β for large β. GOOD.

4.3 Catastrophic Cancellation
Interval-censored probability computed as R(left)-R(right) instead of
CDF(right)-CDF(left). This avoids cancellation when both CDF values are
near 1. CORRECT professional implementation.

4.4 AD Statistic — Fi Clamping
Fi values clamped to [1e-15, 1-1e-15] to avoid log(0). CORRECT.

4.5 Log-Space Hessian
Fisher CI computed in (log β, log η) space, reducing numerical issues from
the curvature of the Weibull likelihood surface. GOOD practice.


5. INPUT PARSING AND DATA VALIDATION
--------------------------------------

5.1 Number Parser (parseNumber, lines ~1686-1733)
Handles: US (1234.56), EU (1234,56), EU thousands (1.234,56), US thousands
(1,234.56), French spaces (1 234,56).

MAJOR ISSUE MAJ3: The disambiguation between "1,234" (US thousands = 1234)
and "1,234" as an EU decimal (1.234) uses the heuristic: "exactly 3 digits
after comma → US thousands." This heuristic fails for values like "1,500"
(which would be parsed as 1500 instead of 1.5 in EU locale), or "2,000"
(parsed as 2000 instead of 2.0). In a reliability engineering context where
both European and US engineers may use the tool, this ambiguity could silently
produce wrong data without any warning. A locale selection dropdown or an
explicit user declaration of decimal separator would be more robust.

5.2 Interval Pair Parser (parseIntervalPairs, lines ~1745-1758)
Regex: /^(\d+(?:\.\d+)?(?:[eE][+-]?\d+)?)\s*-\s*(...)$/
Handles integers, decimals, and scientific notation. Validates left>0 and
right>left. CORRECT.

5.3 Input Validation
- Drops values ≤ 0 (correct for failure times)
- Warns on dropped/NaN values
- Requires ≥ 2 exact failures for 2P, ≥ 3 for 3P
- Detects degenerate data (CV < 0.001, all times nearly identical)
- All validations trigger alerts or toasts with informative messages
VERDICT: ADEQUATE. Validation is thorough.

5.4 Excel Import
Loads last row from first sheet. Column detection is case-insensitive with
fallbacks (Failures/failures, Suspensions/suspensions, etc.). ADEQUATE.

OBSERVATION O8: Excel import reads only the LAST row of data. This design
choice (documented in the comment "Use the LAST row (most recent analysis)")
is appropriate for round-tripping exported analysis results, but could
surprise users who place raw data in multiple rows expecting row-by-row import.
A clearer UI note would prevent confusion.


6. EDGE-CASE ANALYSIS
-----------------------

Case 1: n=2 (minimum exact failures)
  - KS and AD explicitly disabled (correct)
  - MLE runs but produces wide, unreliable intervals (expected)
  - Warning displayed. HANDLED.

Case 2: 100% censored (no exact failures)
  - Caught by "exactObs.length < 2" check → alert. HANDLED.

Case 3: All identical failure times
  - Caught by checkDegenerateData (CV < 0.001) → alert. HANDLED.

Case 4: Single suspension + n exact failures
  - Herd-Johnson computes correctly (tested manually). HANDLED.

Case 5: Very large β (β → ∞, near-deterministic failures)
  - Chart rendering clamped at β=50 with warning. HANDLED.

Case 6: γ → min(t) in 3P MLE
  - Likelihood → ∞, not explicitly guarded; golden-section may converge
    to a boundary near min(t) with very high ll. Not caught with a dedicated
    warning. PARTIALLY HANDLED (see OBSERVATION O1).

Case 7: Interval where left ≥ right
  - Filtered out by parseIntervalPairs (right > left enforced). HANDLED.

Case 8: Negative input values
  - Dropped by filter(v => v > 0). HANDLED.

Case 9: Mixed interval + exact + right-censored data
  - All three observation types combined in the observations array and

  - Detected (arr.length < 10 → 'insufficient resamples'). HANDLED.

12. RECOMMENDATIONS AND REMEDIATION
--------------------------------------

Priority 1 — Address MAJ3 (Number Parser):
  Add an explicit locale selector (US/EU/Auto) to the input form, or use
  browser's Intl.NumberFormat for locale-aware parsing. At minimum, display
  the parsed numerical values back to the user before analysis to allow
  verification.

Priority 2 — Address MAJ1 (Bootstrap Interval Data):
  For interval-censored observations, sample a latent failure time from the
  truncated Weibull CDF within [left, right] using inverse CDF:
    t_lat = gamma + eta * (-ln(1 - (F(left) + U*(F(right)-F(left)))))^(1/beta)
  This adds two lines per resample and is the statistically correct approach.

Priority 3 — Address MAJ2 (KS/AD Anti-Conservative p-values):
  Implement Lilliefors-style corrected critical values for the Weibull KS
  test (tabulated in Stephens 1977 or Lockhart & Stephens 1994). If not
  feasible, add a clearer visual disclaimer adjacent to the p-values themselves
  (not just in the GoF block) stating "These p-values are NOT valid for
  comparing against standard α thresholds."

Priority 4 — Improve M2 (Regression CIs):
  Replace the heuristic regression CI with the proper standard error of the
  OLS slope: SE(β̂) = √(σ²_ε / SSxx), where σ²_ε = SSres/(n-2). This is a
  standard OLS formula requiring only a few additional lines.

Priority 5 — Address M7 (Bias Correction Label):
  Update the checkbox label from "Thoman et al. 1969" to "Ross (1994)
  approximation (Thoman 1969 basis)" and optionally provide the original
  tabulated correction factors for complete accuracy.

Priority 6 — Address O1 (Unbounded 3P Likelihood):
  Detect when γ converges to within 1% of min(t) and display a prominent
  warning that the 3P MLE may be ill-conditioned (likelihood asymptotically
  unbounded).

Priority 7 — Address O5 (Bootstrap Reproducibility):
  Add a text field for a random seed, using a seeded PRNG (e.g., xoshiro256
  in JavaScript) instead of Math.random() when a seed is provided.

Priority 8 — Address O9 (Chart.js Version):
  Pin the Chart.js CDN URL to a specific version, e.g.:
  https://cdn.jsdelivr.net/npm/chart.js@4.4.3/dist/chart.umd.min.js

Priority 9 — Address M3 (3P Regression Refinement):
  Apply golden-section refinement after the CCO grid, mirroring the 3P MLE
  approach. This requires ~15 additional function evaluations and takes
  negligible time.


13. CONCLUSION
--------------
The Weibull Analysis Tool (Professional v8) is a well-engineered, mathematically
sound browser application for reliability data analysis. Its core mathematical
implementations are correct and traceable to the primary statistical literature
(Meeker & Escobar, Abernethy, Nelson, Lawless). Results agree with R, Python
scipy, Minitab, and ReliaSoft Weibull++ within expected numerical precision
on all reference datasets tested.

Notable strengths:
  + Profile likelihood CIs correctly implemented (superior to Fisher alone)
  + Boundary-corrected LR test (Self & Liang 1987) — rare in non-commercial tools
  + Log-parametric Hessian for Fisher CI (numerically superior approach)
  + Herd-Johnson adjusted ranks correctly implemented per Abernethy (2006)
  + Professional handling of interval-censored data (R(left)-R(right) method)
  + Multi-start optimizer strategy for robustness against local minima
  + Explicit acknowledgment of anti-conservative KS/AD bias

Key weaknesses:
  + Bootstrap CI underestimates width with significant interval-censored data
  + Number parser locale ambiguity can silently misparse European decimals
  + KS/AD p-values displayed without Lilliefors correction caveats at point of use
  + Regression CIs use a heuristic formula without statistical derivation

This tool is suitable for professional reliability engineering tasks with
properly formatted (US locale) data and predominantly exact or right-censored
observations. For interval-censored dominated datasets and bootstrap CIs, the
user should apply extra caution and cross-validate with a reference package.

With the remediation of the three major issues identified (particularly the
number parser locale ambiguity and bootstrap interval-censored resampling),
the tool would meet the quality bar of professional-grade reliability analysis
software.
